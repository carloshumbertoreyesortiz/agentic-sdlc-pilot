# Plan — Fix RFC 4180 CSV escaping (seed task US-021)

> Produced by the `/plan` flow (E-05, impl guide §07). Approved to proceed by
> carloshumbertoreyesortiz. Work lands on `agent/csv-escape-fix`.

## Goal
Provide a correct RFC 4180 CSV field-escaping utility so fields containing
commas, double-quotes, or line breaks are serialized safely (quoted, with inner
quotes doubled), plus a row joiner.

## Out of scope
- Full CSV *parsing* / reading.
- Streaming, files, or encodings beyond UTF-8 strings.
- Configurable delimiters (RFC 4180 comma only).

## Files to touch
- `src/csv.ts` — add `escapeCsvField()` and `toCsvRow()` (new).
- `src/csv.test.ts` — unit tests covering the RFC 4180 cases (new).
- `.agent/provenance.json` — machine-generated by `scripts/write-provenance.ts`.

## Test plan
- `escapeCsvField` leaves plain fields unchanged.
- Quotes a field containing a comma, a CR, or an LF.
- Doubles inner `"` and wraps the field.
- `toCsvRow` joins escaped fields with commas.
- Verify with `npm run check` (lint + build + test).

## Acceptance criteria
- [ ] `artifacts/plan.md` written; no source edits before approval.
- [ ] Fix implemented on `agent/csv-escape-fix`.
- [ ] `npm run check` passes.
- [ ] PR carries a machine-generated `.agent/provenance.json` and passes the
      `agent-provenance` gate.

## Risk flags
- Low risk: pure string utility, no I/O, no secrets, no untrusted input at
  runtime. Main correctness risk is mishandling embedded quotes/newlines —
  covered by tests.
